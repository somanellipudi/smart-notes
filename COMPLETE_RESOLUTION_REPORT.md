## THE PROBLEM - IEEE Access Red Flags

### Red Flag 1: Figure 1 Embedded Specs
**Issue:** Compiled paper PDF Figure 1 contained:
- "CalibraTeach: 7-Stage Real-Time Fact Verification Pipeline" (title banner)
- "GPU: NVIDIA RTX 4090" (hardware specs)
- "PyTorch 2.0.1, CUDA 11.8, Transformers 4.35.0" (framework versions)

**Why Critical:** 
- Violates IEEE Access reproducibility guidelines (no system specs in figures)
- Reviewers would flag as missing figure quality standards

### Red Flag 2: Soft-Hyphen Artifacts "￾"
**Issue:** PDF text extraction showed:
- "reproduc￾tion" instead of "reproduction"
- "well￾calibrated" instead of "well-calibrated"
- "confidently" with invisible soft-hyphen in middle

**Why Critical:**
- Makes abstract unreadable when copy-pasted
- Breaks automated text processing by reviewers
- Appears as encoding corruption in databases
- IEEE accepts papers with clean text extraction

### Red Flag 3: No Compiled PDF Validation
**Issue:** validate_paper_ready.py didn't verify the COMPILED PDF
- Only checked source files
- Didn't catch Figure 1 issues in final output
- Couldn't detect soft-hyphen artifacts after compilation

**Why Critical:**
- False confidence: validation reports success but compiled PDF fails
- No early detection: issues found too late
- No fail-fast: tests waste time before catching problems

### Solution 2: Soft-Hyphen Artifacts Mitigated ✅

**Changes to [paper/main.tex](paper/main.tex):**

#### Layer 1: Unicode/Glyph Mapping (Existing)
```latex
% Enable Unicode glyph-to-text mapping in PDF
\IfFileExists{glyphtounicode.tex}{\input{glyphtounicode}}{}
\pdfgentounicode=1
\usepackage{cmap}

% Map soft-hyphen to empty string (prevents "￾" in copy-paste)
\DeclareUnicodeCharacter{00AD}{}
```

#### Layer 2: Global Hyphenation Reduction (NEW)
```latex
% Microtype reduces discretionary hyphenation globally while maintaining typography
% [final] option activates at PDF generation (not draft mode) - IEEE safe
\usepackage[final]{microtype}
```

#### Layer 3: Scoped No-Hyphenation (NEW - Most Critical)
```latex
% Scoped to Abstract+Keywords (most text-extraction-heavy regions)
% Prevents discretionary hyphens in sections copied by academic databases
\begingroup
\hyphenpenalty=10000          % Extreme penalty for hyphenation
\exhyphenpenalty=10000        % Extreme penalty for explicit hyphens

\begin{abstract}
  ...abstract text (no hyphens)...
\end{abstract}

\begin{IEEEkeywords}
  ...keywords (no hyphens)...
\end{IEEEkeywords}

\endgroup

% Document body unaffected - maintains IEEE typography and line breaking
```

**Why layered approach:**
- Layer 1: Catches any existing soft-hyphens (maps to null)
- Layer 2: Prevents new soft-hyphens globally (via microtype)
- Layer 3: Prevents hyphens in most critical sections (abstract+keywords)

### Solution 3: Compiled PDF Validation Script ✅

**New File: [scripts/compile_and_check_pdf.py](scripts/compile_and_check_pdf.py)**

**Workflow:**
```
1. Detect pdflatex on system
   ├─ If found: proceed to compile
   └─ If not: graceful skip with [INFO] message

2. Extract dist/overleaf_submission.zip to temp directory
   └─ Preserves original zip

3. Run pdflatex compilation
   └─ pdflatex -interaction=nonstopmode main.tex

4. Run PDF hygiene check on compiled PDF
   └─ python scripts/check_pdf_text_hygiene.py paper.pdf

5. Clean up temporary directory
   └─ Removes temp files (no artifacts left)
```

**ASCII-Safe Output (Windows cp1252 compatible):**
```
[OK] Found pdflatex: C:\...\pdflatex.exe
[OK] Extracted overleaf_submission.zip to temp\...
[OK] Compiling main.tex...
[OK] PDF compiled: temp\paper.pdf
[RESULT] Hygiene check output:
[OK] No replacement artifacts found
[OK] Architecture PDF contains no embedded specs/titles
[OK] Compilation check structure ready
```

**Error Handling:**
- pdflatex not found → graceful skip (return 0, not error)
- Compilation fails → detected and reported
- Hygiene check fails → return exit code 1 (fail-fast)
- Temp cleanup always runs (even on errors)
